Understanding Transformers (from scratch)

To learn how large language models (like GPT, LamDA, and others) work, it’s useful to understand how transformers work.
To understand how transformers work, it’s good to understand how the attention mechanism works.
To understand how the attention mechanism works, it’s useful to understand how sequence-to-sequence (seq2seq) models work.
To understand how seq2seq models work, it’s useful to understand how recurrent neural networks (RNNs) work.
To have a better understanding of RNNs, one should have a foundational knowledge of neural networks and deep learning.
To learn about neural networks and deep learning, you probably first need to understand some basic machine learning concepts.
— From a Google presentation, go/llms-from-scratch
So let's go over these, bottom to top.

Basic Machine Learning Concepts

How do machines "learn"?

An ML model is basically a very complex mathematical function with [lots of] parameters (aka weights). At the beginning, these weights are usually initialized with random values (or other reasonable guesses). The "learning process" (called training) involves the following high level steps:

  1. Run the ML model (i.e. compute the math function) with a batch of input data, to compute an output
  2. Evaluate the emitted output (known as the loss function)
  3. Update the parameter weights slightly to improve (reduce) the loss
  4. Go back to step 1 (stop when we are satisfied)

The idea is that with sufficient training and data using the above steps, the parameters of the complex mathematical function will converge to good values such that the model is able to predict values accurately on new data. The model learns and improves over time via the training process.

This is in contrast to normal/traditional/non-learning algorithms whose parameters are "fixed". They do not require any training, but they also do not learn from the data. They work great when we already know the computation model (true for the vast majority of code in the current world). But ML models have been able to solve a variety of problems that we don't have an intuitive sense of how to model the problem.

The "XKCD: Is it a bird?" problem was considered a very hard problem not so long ago, but has become easy to solve via ML approaches.
